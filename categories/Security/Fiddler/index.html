Man-in-the-middle and the JavaScript Keylogger

When a website is HTTPS enabled, it ensures that your communication to the website is encrypted. It does not ensure that the website is completely safe (as there may be other vulnerabilities in the site), but it protects against man-in-the-middle (MITM) attacks.

Although MITM attacks look very unlikely to happen, but they are very likely. This blog post is a simple security experiment to demonstrate a scenario where a user falls prey to MITM attack and loses his/her password.

Take a look at how MITM works:

How does the hacker intercept the traffic of the user? It could be through software such as malware which was spread to the victim or even through tapping into hardware such as the switch or router the victim connects to. In our example, we take the all too familiar case where the victim uses a proxy server found online to bypass the network restrictions. This is common in workplaces, schools and colleges where a lot of popular sites are made inaccessible by the network admin. In such cases, users usually search online for a proxy server and connect to it. In our scenario the proxy server is owned by the hacker:

A lot of sites have a http homepage, but have a link on their homepage for login that takes the user to a https enabled page that asks for credentials (Example - amazon.com). What’s wrong with this approach is that the hacker who can intercept the homepage can also add a prompt for the user’s credentials at the homepage itself. The login prompt can also be specifically crafted with CSS that matches the website, so as not to look suspicious.

If you look at step (4) in the above picture, the hacker sends back the requested webpage to the user with some malicious code attached. In our scenario, we attach two things to the page:
1) A javascript keylogger that sends every keypress made on the page to a webservice that is somewhere hosted by the hacker.
2) A login prompt (if not already present in the page) asking the user to enter the credentials to the site.
